# Development Rules вЂ” OECM Favourability Tool Project

## Scientific Context
This project implements a multi-criteria analysis (MCE) tool to identify
territories favourable for OECMs (Other Effective Area-based Conservation
Measures, CBD COP14 decision 14/8). Modelling decisions are documented
in SPECIFICATIONS.md.

## Code Rules

### General
- Python в‰Ґ 3.10
- All analytical functions have complete docstrings (numpy style)
- Explicit handling of raster NoData values (np.nan, rasterio nodata)
- Scientific parameters (thresholds, default weights) are defined
  exclusively in config/ вЂ” never hardcoded
- Logging via the standard logging module, INFO level by default

### Raster Processing
- Reference CRS: ETRS89 / EPSG:3035 for Europe (adaptable per territory)
- Reference resolution: user-defined in config/settings.yaml
- Resampling: resampling.Resampling.bilinear by default (configurable)
- All layers are aligned to the same grid before any calculation

### Analytical Modules (DO NOT modify without validation)
- The aggregation method is Yager OWA OR weighted geometric mean
  вЂ” WLC (linear combination) is FORBIDDEN as the main method
- Functional groups (D, A, B, C) are defined in SPECIFICATIONS.md
  and must not be restructured without explicit instruction
- The anthropogenic pressure layer plays a dual role (see SPECIFICATIONS.md)
- Provisioning ES capacity uses a Gaussian function (non-monotone)

### Streamlit Interface
- Strict separation between business logic (modules/) and interface (ui/)
- Scientific parameters modified in the interface are always
  saveable and exportable (reproducibility)
- The ОЈ weights = 1 constraint is verified and displayed in real time

### Tests
- Unit tests in tests/ for all analytical modules
- Synthetic data generated by tests/generate_synthetic_data.py
- Never use real data in automated tests

### Configuration
- All parameters in config/ (YAML)
- No scientific values hardcoded in source code
- The CLC/OSO в†’ group compatibility table is in config/land_use_compatibility.yaml
